# AZ-500 Azure Security Study and Practice Summary

## Overview

This project summarizes several Azure security labs I completed previously as part of my AZ-500 study and hands-on practice.

The work was based on the Microsoft Learning AZ-500 lab repository and focused on practical exposure to Azure identity, access control, network restrictions, infrastructure security, and administrative security tasks.

## Source

The exercises were based on the official Microsoft Learning repository:

- MicrosoftLearning/AZ500-AzureSecurityTechnologies

This summary documents the areas I practiced and the security concepts I worked with, rather than reproducing the original lab instructions.

## Lab Scope

I completed several labs from the 02–07 portion of the Microsoft Learning AZ-500 materials.

My work in those labs included hands-on practice in areas such as:

- identity and access management
- Azure administrative security tasks
- SQL connectivity and access scenarios
- network rules and protocol control on infrastructure resources
- validating access to servers through opened ports
- restricting one virtual machine from reaching another
- connecting SQL-related services to virtual machine environments when needed
- working with user access and IAM-related configuration

## What I Practiced

Through this work, I practiced:

- Azure IAM concepts
- role-based access and user-related administration
- network access control in Azure environments
- security-focused infrastructure configuration
- server access validation through controlled ports
- segmentation and communication restriction between systems
- connecting application or database components in a controlled environment

## What I Learned

- Identity and access management is closely tied to practical infrastructure security
- Network rules and protocol control are important parts of securing IaaS-style environments
- Azure security work often requires validating both connectivity and restriction scenarios
- Access to services such as SQL should be configured carefully and with awareness of user permissions, exposure, and operational need
- Hands-on labs are useful for understanding how Azure security concepts work beyond theory

## Tools and Platforms Involved

- Microsoft Azure
- Microsoft Entra ID
- Azure virtual machines
- Azure network security concepts
- SQL connectivity scenarios
- Azure administrative and access-control configuration
